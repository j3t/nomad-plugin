This is the Job definition used to spawn a new agent in Nomad. You can customize it but two important things have to be kept in mind:
  1. Job.ID must be %WORKER_NAME%
  2. JNLP agent must be started up and the agent name must be %WORKER_NAME%
Below you can find a list of placeholders which are resolved before the job is being sent to Nomad.
  • %WORKER_NAME% - name of the Jenkins agent (e.g. jenkins-146ddad3c95c9).
  • %WORKER_SECRET% - secret for this agent to connect to Jenkins (e.g. 7f1bc810deeba3a4624aea3ef0fbc74b6d56bdee27ea006a1c8f560aa66075d4).
Note: The default job definition is pretty much ready to go. You only have to adjust Region, Datacenters, JENKINS_URL and JENKINS_TUNNEL.
